How It Works

Each pixel is modelled as a mixture of K Gaussians. Every frame, the algorithm checks whether the current pixel value matches any existing component (within T standard deviations). Matched components update their mean, variance, and weight using learning rate α.

Components are ranked by ω/√σ² — high weight, low variance means a stable background. The top components whose weights sum to B form the background model. Pixels that don't match any background component are classified as foreground.

When no component matches a pixel, the weakest component is replaced with a new Gaussian centred on the current value — allowing the model to adapt to entirely new background regions.
